# SmartTScope — Quickstart Guide

**Summary**: Step-by-step guide for first-time setup on Raspberry Pi 5 running Raspberry Pi OS Trixie (Debian 13).

**Sources**: install_pi.sh, hardware-platform.md, scripts/install_pi.sh

**Last updated**: 2026-04-30

---

## Platform

| Component | Specification |
|---|---|
| Board | Raspberry Pi 5 (4 GB or 8 GB) |
| OS | Raspberry Pi OS 64-bit — **Debian 13 "Trixie"** |
| Camera | ToupTek toupcam USB (SDK via `toupcam` Python package) |
| Mount | OnStep V4 via USB serial (LX200 protocol) |
| Focuser | OnStep V4 focuser channel (shared serial port) |
| Plate solver | ASTAP with G17 (D80) star catalog |
| Python | 3.13 (available natively in Trixie repos) |

> **libcamera is not used.** SmartTScope uses the ToupTek `toupcam` SDK directly over USB, not the Pi's built-in camera stack. Do not install or configure libcamera.

## First-time installation

```bash
# 1. Clone the repository
git clone https://github.com/tschoenfelder/SmartTScope.git
cd SmartTScope

# 2. Run the installer (installs Python 3.13, venv, all deps)
bash scripts/install_pi.sh

# 3. Optional: also install ASTAP plate solver
bash scripts/install_pi.sh --with-astap
```

Python 3.13 is available directly from the Trixie apt repository — no PPA or deadsnakes required.

## ASTAP catalog

Download the G17 (D80) star catalog from [hnsky.org](https://www.hnsky.org/astap.htm) and extract the `.290` files to `~/.astap/`. The server reports `solver: error` at `POST /api/session/connect` if the catalog is missing.

## Start the server

```bash
source .venv/bin/activate
uvicorn smart_telescope.app:app --host 0.0.0.0 --port 8000
```

Open `http://<pi-address>:8000` in a browser to access the control panel.

## Environment variables

| Variable | Default | Description |
|---|---|---|
| `ONSTEP_PORT` | — | Serial port for the OnStep mount (e.g. `/dev/ttyUSB0`). If unset, mock adapters are used. |
| `SIMULATOR_FITS_DIR` | — | Directory of FITS files for the SimulatorCamera (replay mode). Used when no `ONSTEP_PORT`. |
| `OBSERVER_LAT` | `50.336` | Observer latitude in decimal degrees (default: Usingen, Germany) |
| `OBSERVER_LON` | `8.533` | Observer longitude in decimal degrees |
| `STARS_CFG` | `stars.cfg` | Path to the custom target file (TOML format) |
| `PIXEL_SCALE_ARCSEC` | `0.38` | Pixel scale hint for the plate solver (C8 native focal length) |

## Custom targets

Edit `stars.cfg` to add your own targets in TOML format. The server reloads the file on every request — no restart needed. Available via `GET /api/catalog/stars`.

## Systemd auto-start

```bash
sudo cp scripts/smarttscope.service /etc/systemd/system/
sudo systemctl enable --now smarttscope
```

(The service file is generated by `install_pi.sh --with-astap`.)
